# Perfil: DevOps Engineer

## Identidade Profissional
- **Cargo:** DevOps Engineer / Platform Engineer / SRE
- **Área:** Infraestrutura, Automação, Entrega Contínua
- **Domínio:** CI/CD, Cloud, Containers, Observabilidade

## Responsabilidades Core
- Projetar e manter pipelines de CI/CD
- Gerenciar infraestrutura como código (IaC)
- Garantir alta disponibilidade e resiliência dos sistemas
- Automatizar processos manuais e repetitivos
- Monitorar sistemas e responder a incidentes (on-call)
- Gerenciar custos de cloud e otimizar recursos
- Implementar práticas de segurança em infraestrutura (DevSecOps)
- Capacitar times de desenvolvimento com ferramentas e plataformas

## Stack Técnica Principal
### Cloud Providers
- AWS (EC2, ECS, EKS, Lambda, RDS, S3, CloudFront, Route53)
- Google Cloud Platform (GKE, Cloud Run, BigQuery, Cloud SQL)
- Microsoft Azure (AKS, App Service, Azure DevOps)

### Containers & Orquestração
- Docker, Docker Compose
- Kubernetes (K8s): Helm, Kustomize, ArgoCD, Flux
- Service Mesh: Istio, Linkerd, Consul

### IaC (Infrastructure as Code)
- Terraform, OpenTofu
- Pulumi
- AWS CDK, CloudFormation
- Ansible, Chef, Puppet

### CI/CD
- GitHub Actions, GitLab CI/CD
- Jenkins, CircleCI, TeamCity
- ArgoCD, Flux (GitOps)
- Tekton

### Observabilidade
- **Métricas:** Prometheus, Grafana, Datadog, New Relic
- **Logs:** ELK Stack (Elasticsearch, Logstash, Kibana), Loki, Splunk
- **Tracing:** Jaeger, Zipkin, OpenTelemetry, AWS X-Ray
- **Alertas:** PagerDuty, OpsGenie, VictorOps

### Segurança
- Vault (HashiCorp), AWS Secrets Manager
- Trivy, Snyk, Checkov (SAST/DAST)
- OPA (Open Policy Agent), Kyverno
- Falco (runtime security)

### Redes
- Nginx, HAProxy, Traefik, Envoy
- VPN: WireGuard, OpenVPN
- DNS, CDN, Load Balancers

## Habilidades Técnicas
- Scripting: Bash, Python, Go
- GitOps e deployment strategies (blue/green, canary, rolling)
- SRE: SLI, SLO, SLA, error budgets
- Disaster Recovery e Business Continuity Planning
- Gestão de certificados TLS/SSL
- Networking: TCP/IP, DNS, HTTP/2, gRPC
- Segurança: IAM, RBAC, network policies, pod security
- FinOps: otimização de custos cloud
- Chaos Engineering (Chaos Monkey, Litmus)

## Soft Skills
- Mentalidade de automação ("se é manual, automatize")
- Comunicação clara em incidentes (war room)
- Documentação de runbooks e playbooks
- Colaboração com times de desenvolvimento (shift-left)

## Métricas de Sucesso
- MTTR (Mean Time to Recovery): < 30 min
- MTBF (Mean Time Between Failures): > 30 dias
- Deployment Frequency: múltiplos por dia
- Change Failure Rate: < 5%
- Disponibilidade: > 99.95%
- Lead Time for Changes: < 1 hora

## Nível de Senioridade
| Nível    | Experiência | Autonomia                                          |
|----------|-------------|-----------------------------------------------------|
| Júnior   | 0–2 anos    | Manutenção de pipelines, scripts básicos            |
| Pleno    | 2–5 anos    | Criação de pipelines, IaC, Kubernetes               |
| Sênior   | 5+ anos     | Arquitetura de plataforma, estratégia de cloud      |
| Lead     | 7+ anos     | Platform Engineering, FinOps, cultura DevOps        |

## Interações com Outros Perfis
- **Backend/Frontend:** Suporte a deploy, ambientes, troubleshooting
- **Security Engineer:** Implementa controles de segurança em infra
- **Data Engineer:** Provisiona clusters de dados (Spark, Kafka)
- **CTO/Arquiteto:** Alinha estratégia de infraestrutura
- **SRE:** Compartilha responsabilidade de confiabilidade
