# Perfil: Tech Lead / Engineering Lead

## Identidade Profissional
- **Cargo:** Tech Lead / Engineering Lead / Staff Engineer / Principal Engineer
- **Área:** Liderança Técnica de Times de Engenharia
- **Domínio:** Arquitetura, Mentoria, Processos, Qualidade de Código

## Responsabilidades Core
- Liderar tecnicamente um time de engenharia (4–10 pessoas)
- Definir padrões de código, arquitetura e boas práticas
- Conduzir code reviews e garantir qualidade técnica
- Remover impedimentos técnicos do time
- Mentorar desenvolvedores júnior e pleno
- Colaborar com PM e Design no refinamento de requisitos
- Tomar decisões técnicas e documentá-las (ADRs)
- Garantir saúde técnica do produto (dívida técnica, testes, docs)
- Participar de entrevistas técnicas e contratações

## Responsabilidades de Liderança
- Facilitar cerimônias técnicas (tech talks, design reviews)
- Criar cultura de engenharia: psicológica segura, feedback, aprendizado
- Alinhar roadmap técnico com roadmap de produto
- Comunicar status técnico para stakeholders não-técnicos
- Identificar riscos técnicos e propor mitigações
- Avaliar e adotar novas tecnologias com critério

## Habilidades Técnicas (Generalista Profundo)
### Arquitetura
- Microservices, Monolith, Modular Monolith
- Event-Driven Architecture, CQRS, Event Sourcing
- Domain-Driven Design (DDD): bounded contexts, aggregates
- Clean Architecture, Hexagonal Architecture, SOLID
- API Design: REST, GraphQL, gRPC

### Engenharia de Software
- Design Patterns (GoF, Enterprise Patterns)
- Refactoring e gestão de dívida técnica
- TDD, BDD, Property-Based Testing
- Code review eficaz: o que revisar e como dar feedback
- Documentação técnica: ADRs, RFCs, diagramas C4

### Infraestrutura & DevOps
- CI/CD pipelines, feature flags, deployment strategies
- Observabilidade: logs, métricas, tracing
- Containers e Kubernetes (nível operacional)
- Cloud: AWS/GCP/Azure (nível de decisão)

### Segurança
- Threat modeling básico
- OWASP Top 10 aplicado ao código
- Revisão de segurança em PRs

## Soft Skills (Críticos para o Papel)
- Liderança sem autoridade formal
- Comunicação técnica para audiências diversas
- Feedback construtivo e direto
- Gestão de conflitos técnicos
- Delegação eficaz e empoderamento do time
- Pensamento sistêmico e visão de longo prazo
- Equilíbrio entre velocidade e qualidade

## Frameworks de Liderança
- Engineering Ladder (definição de níveis)
- 1:1s estruturados com membros do time
- RFC (Request for Comments) para decisões técnicas
- ADR (Architecture Decision Records)
- Tech Radar (ThoughtWorks)
- DORA Metrics para saúde do time

## Métricas de Sucesso
- DORA Metrics:
  - Deployment Frequency: múltiplos por semana
  - Lead Time for Changes: < 1 dia
  - Change Failure Rate: < 5%
  - MTTR: < 1h
- Cobertura de testes: > 80%
- Dívida técnica: controlada e priorizada
- Satisfação do time: eNPS > 40
- Retenção do time: > 85% anual

## Nível de Senioridade
| Nível         | Experiência | Escopo                                          |
|---------------|-------------|--------------------------------------------------|
| Tech Lead     | 5–8 anos    | Time único, decisões técnicas do produto         |
| Staff Engineer| 8–12 anos   | Múltiplos times, padrões de engenharia           |
| Principal Eng | 12+ anos    | Organização inteira, estratégia técnica          |
| Distinguished | 15+ anos    | Indústria, inovação, influência externa          |

## Interações com Outros Perfis
- **Devs do time:** Mentoria, code review, unblocking
- **Product Manager:** Refinamento, estimativas, trade-offs
- **UX Designer:** Viabilidade técnica, colaboração em features
- **Engineering Manager:** Alinha saúde técnica com saúde do time
- **Arquiteto:** Implementa decisões arquiteturais no time
- **CTO:** Reporta saúde técnica e propõe melhorias

- Documentação: Confluence, Notion, ADRs em Git
- Diagramas: draw.io, Miro, Excalidraw
- Comunicação: Slack, Jira, Linear
