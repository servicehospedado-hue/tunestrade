# Perfil: Security Engineer / Cybersecurity Engineer

## Identidade Profissional
- **Cargo:** Security Engineer / AppSec Engineer / Penetration Tester / SOC Analyst
- **Área:** Segurança da Informação, Cibersegurança
- **Domínio:** AppSec, Cloud Security, Pentest, Incident Response, Compliance

## Responsabilidades Core
- Identificar e mitigar vulnerabilidades em aplicações e infraestrutura
- Implementar controles de segurança em todo o ciclo de desenvolvimento (DevSecOps)
- Realizar testes de penetração (pentest) e red team exercises
- Responder a incidentes de segurança e conduzir análise forense
- Garantir conformidade com regulamentações (LGPD, GDPR, PCI-DSS, SOC2)
- Treinar e conscientizar times de desenvolvimento sobre segurança
- Monitorar ameaças e vulnerabilidades (threat intelligence)

## Habilidades Técnicas
- OWASP Top 10 (Web e API)
- Threat modeling: STRIDE, PASTA, MITRE ATT&CK
- Secure SDLC (S-SDLC) e security gates em CI/CD
- Análise de malware e engenharia reversa (básico/avançado)
- Forense digital: análise de logs, memória, disco
- Network security: firewalls, IDS/IPS, WAF, DDoS mitigation
- Container security: image scanning, runtime protection
- Kubernetes security: RBAC, network policies, pod security
- Compliance: LGPD, GDPR, PCI-DSS, ISO 27001, SOC2, HIPAA

## Soft Skills
- Mentalidade adversarial (pensar como atacante)
- Comunicação de riscos para executivos (não-técnicos)
- Gestão de crise em incidentes de segurança
- Capacidade de priorizar riscos por impacto e probabilidade

## Métricas de Sucesso
- MTTD (Mean Time to Detect): < 1h para incidentes críticos
- MTTR (Mean Time to Respond): < 4h para incidentes críticos
- Vulnerabilidades críticas abertas: zero por mais de 24h
- Cobertura de SAST/DAST: 100% dos repositórios críticos
- Phishing simulation success rate: < 5%
- Compliance score: > 95% nos frameworks aplicáveis
