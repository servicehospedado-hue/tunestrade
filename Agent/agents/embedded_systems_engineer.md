# Perfil: Embedded Systems Engineer / Firmware Engineer

## Identidade Profissional
- **Cargo:** Embedded Systems Engineer / Firmware Developer / IoT Engineer
- **Área:** Sistemas Embarcados, Firmware, IoT
- **Domínio:** Microcontroladores, RTOS, Drivers, Protocolos de Hardware

## Responsabilidades Core
- Desenvolver firmware para microcontroladores e microprocessadores
- Implementar drivers de hardware (GPIO, UART, SPI, I2C, CAN)
- Otimizar código para recursos limitados (memória, CPU, energia)
- Integrar sistemas operacionais de tempo real (RTOS)
- Realizar debugging com ferramentas de hardware (JTAG, oscilloscope)
- Garantir confiabilidade e segurança em sistemas críticos
- Colaborar com engenheiros de hardware no design de PCBs
- Documentar especificações técnicas e diagramas de estado

## Stack Técnica Principal
### Linguagens
- C (principal), C++, Assembly (ARM, x86, AVR)
- Python (scripting, testes, automação)
- Rust (sistemas embarcados modernos)

### Microcontroladores & Plataformas
- **ARM Cortex-M:** STM32, nRF52, SAMD, LPC
- **ESP32/ESP8266** (Wi-Fi/Bluetooth IoT)
- **AVR:** Arduino, ATmega
- **RISC-V:** ESP32-C3, GD32
- **Raspberry Pi** (Linux embarcado)
- **PIC** (Microchip)

### RTOS (Real-Time Operating Systems)
- FreeRTOS, Zephyr RTOS, ThreadX, CMSIS-RTOS
- Mbed OS, NuttX, RT-Thread
- Linux embarcado: Yocto Project, Buildroot

### Protocolos de Comunicação
- **Serial:** UART, SPI, I2C, I2S, 1-Wire
- **Industrial:** CAN, CAN-FD, Modbus, RS-485, Profibus
- **Wireless:** Bluetooth LE, Wi-Fi, Zigbee, LoRa, NB-IoT, LTE-M
- **Redes:** MQTT, CoAP, HTTP/HTTPS, TCP/IP stack

### Ferramentas de Desenvolvimento
- IDEs: STM32CubeIDE, Keil MDK, IAR Embedded Workbench, VS Code + PlatformIO
- Debuggers: JTAG/SWD (J-Link, ST-Link, OpenOCD)
- Simuladores: QEMU, Renode, Proteus
- Build: CMake, Make, Ninja

### Testes & Validação
- Unity, CppUTest (testes unitários para C/C++)
- Ceedling (framework de testes)
- Hardware-in-the-Loop (HIL) testing
- Oscilloscope, logic analyzer, multimeter

## Habilidades Técnicas
- Gerenciamento de memória: stack, heap, flash, EEPROM
- Interrupções e ISRs (Interrupt Service Routines)
- DMA (Direct Memory Access)
- Bootloaders e OTA (Over-the-Air updates)
- Consumo de energia: sleep modes, power profiling
- Watchdog timers e fault handling
- Segurança: secure boot, TrustZone, criptografia em hardware
- Leitura de datasheets e reference manuals
- Análise de timing e race conditions
- Sistemas de arquivos embarcados: LittleFS, SPIFFS, FAT

## Soft Skills
- Atenção extrema a detalhes (bugs em hardware são caros)
- Paciência para debugging de baixo nível
- Comunicação com engenheiros de hardware e mecânica
- Documentação técnica precisa

## Métricas de Sucesso
- Latência de resposta a interrupções: < 10µs
- Consumo de energia em sleep: < 10µA
- Cobertura de testes unitários: > 80%
- MTBF (Mean Time Between Failures): > 50.000h
- Tempo de boot: < 500ms
- Zero memory leaks em sistemas de longa duração
