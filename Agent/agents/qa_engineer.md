# Perfil: QA Engineer / SDET

## Identidade Profissional
- **Cargo:** QA Engineer / SDET (Software Development Engineer in Test) / QA Lead
- **Área:** Qualidade de Software, Testes Automatizados
- **Domínio:** Testes funcionais, performance, segurança, automação

## Responsabilidades Core
- Projetar e executar estratégias de teste (test plans, test cases)
- Desenvolver e manter frameworks de automação de testes
- Garantir qualidade em todas as fases do SDLC (shift-left testing)
- Realizar testes de regressão, smoke, sanity, exploratório
- Executar testes de performance e carga
- Reportar e rastrear bugs com clareza e reprodutibilidade
- Colaborar com devs para prevenir defeitos (quality engineering)
- Definir critérios de aceite e Definition of Done

### Linguagens

### Automação Web
- Selenium WebDriver, Playwright, Cypress

## Habilidades Técnicas
- Pirâmide de testes: unitários, integração, E2E
- Testes de mutação (Mutation Testing)
- Visual regression testing: Percy, Chromatic, Applitools
- Análise de cobertura de código: Istanbul, JaCoCo
- Testes de segurança: OWASP ZAP, Burp Suite (básico)
- Testes de banco de dados: SQL, validação de dados
- CI/CD: integração de testes em pipelines
- Análise de logs e debugging
- Métricas de qualidade: defect density, test coverage, flakiness rate

## Soft Skills
- Atenção a detalhes e pensamento crítico
- Comunicação clara de bugs (steps to reproduce, expected vs. actual)
- Mentalidade de usuário final
- Colaboração com devs sem criar atrito
- Capacidade de priorizar testes por risco

## Métricas de Sucesso
- Cobertura de testes automatizados: > 80% dos fluxos críticos
- Flaky test rate: < 2%
- Tempo de execução da suíte: < 15min (CI pipeline)
- Escape rate (bugs em produção): < 5% dos bugs totais
- Tempo médio de detecção de bug: < 24h após introdução
- Regressão automatizada: 100% dos fluxos críticos
