Problema Identificado:
  Múltiplas vulnerabilidades de segurança básicas presentes na API:
  JWT com secret hardcoded, CORS aberto, sem proteção brute force,
  sem headers de segurança, API key em query params públicos.

Solução Implementada:
  Implementação incremental de cada camada sem quebrar funcionalidade existente.
  Fallbacks seguros para compatibilidade com ambiente de desenvolvimento.

Riscos Identificados:
  - [RISCO_1] LoginBruteForceGuard em memória — reiniciar servidor reseta contadores
    Mitigação: Suficiente para produção inicial; Redis pode ser adicionado depois
  - [RISCO_2] CORS_ORIGINS não definido → ainda usa * em dev
    Mitigação: Warning no log; documentado no .env

Lições Aprendidas:
  - [LIÇÃO_1] Sempre usar secrets.token_hex() para chaves criptográficas
  - [LIÇÃO_2] Headers de segurança HTTP são middleware de uma linha — sem desculpa para não ter
  - [LIÇÃO_3] Brute force guard simples em memória é suficiente para MVP
