=== CACHE DE DYNAMICSTRATEGY ===
Problema: _get_user_custom_strategy fazia query no banco a cada ciclo × cada asset
  1000 usuários × 10 assets = 10.000 queries/ciclo de 5s

Solução: cache de 30s por usuário em _user_strategy_cache
  - Primeira chamada: query no banco, instancia DynamicStrategy, salva no cache
  - Chamadas seguintes (dentro de 30s): retorna instância do cache
  - Cache invalidado automaticamente ao: update, toggle, delete de estratégia

Novo método: invalidate_user_strategy_cache(user_id)
  - Chamado pelas rotas PUT, PATCH /toggle, DELETE de user_strategies
  - factory.py injeta strategy_manager nas rotas via set_strategy_manager()
