=== PROBLEMA 2: "Nenhum usuário conectado" ao usar estratégia pessoal ===
DIAGNÓSTICO:
- autotrade_manager.log: "Full refresh: 0 ativos" — nenhum usuário com autotrade=1 no banco
- toggleStrategy() chamava api.toggleAutotrade(true, target.name) passando o NOME da estratégia pessoal
  (ex: "Minha Estratégia") como strategy_id
- autotrade_service._name_map só conhece 'scalping_5s' e 'trend_m1'
- Nome literal "Minha Estratégia" era salvo no campo strategy_name do banco
- strategy_manager tentava instanciar estratégia com esse nome → falha silenciosa
- Resultado: autotrade=1 salvo mas WS nunca conectava corretamente

CORREÇÃO: HomeScreen.tsx — toggleStrategy() agora passa 'scalping_5s' ao chamar toggleAutotrade
  ANTES: api.toggleAutotrade(true, target.name)
  DEPOIS: api.toggleAutotrade(true, 'scalping_5s')

FLUXO CORRETO APÓS CORREÇÃO:
1. App liga estratégia pessoal → chama toggleAutotrade(true, 'scalping_5s')
2. Backend salva strategy_name='Scalping5s' + autotrade=1 no banco
3. AutotradeManager detecta autotrade=1 → dispara _on_autotrade_enabled
4. Engine conecta WS do usuário via AutotradeManager.connect_user()
5. strategy_manager._execute_for_active_users() encontra o usuário ativo
6. _get_user_custom_strategy(user_id, 'Scalping5s') busca estratégia pessoal ativa do usuário
7. Instancia ScalpingStrategy com parâmetros customizados do banco
8. Trades executam com a estratégia personalizada do usuário

ARQUIVO: aplicativo/TunesTrade2/src/screens/HomeScreen.tsx
